# DontShop - 기능요구사항 명세서

## 프로젝트 개요
**DontShop**은 충동구매를 방지하고 쇼핑 앱 사용을 효과적으로 관리하는 안드로이드 애플리케이션입니다. 사용자의 소비 습관을 제어하고 재정 건강을 개선하기 위한 통합 쇼핑 관리 플랫폼입니다.

---

## 1. 대시보드 (Dashboard)

### 1.1 통계 화면
- **충동구매 차단 통계**
  - 총 차단 횟수 표시
  - 총 절약 금액 표시
  - 차단 성공률(%) 표시
  - 본 달 누적 기록 수

- **시각화**
  - 차트/그래프로 시간대별 충동구매 시도 분포
  - 주간 추세 표시
  - 카테고리별 차단 현황

### 1.2 명언 및 다짐 섹션
- **일일 명언 (Wise Quote)**
  - 한글/영문 명언 데이터베이스 (10개 이상)
  - 매일 자동 갱신
  - 수동 회전(rotate) 기능
  - 터치하여 새로운 명언 로드
  
- **지원 언어**
  - 한국어 명언
  - 영어 명언
  - 시스템 로케일 기반 자동 선택

## 2. 쇼핑 앱 제어 (App Control)

### 2.1 쇼핑 앱 관리
- **앱 추가 기능**
  - 앱 이름 입력
  - 일일 사용 제한 시간 설정 (분 단위)
  - 앱 카테고리 선택 (일반 쇼핑, 패션, 해외직구 등)
  - 패키지명 자동/수동 설정
  - 한 번에 여러 앱 추가 지원

- **앱 목록 표시**
  - 추가된 쇼핑 앱 리스트
  - 각 앱별 정보
    - 앱 이름
    - 일일 제한 시간
    - 오늘 사용한 시간
    - 잠금 상태
    - 카테고리

### 2.2 사용 시간 관리
- **실시간 추적**
  - 쇼핑 앱 사용 시간 실시간 계산
  - 2초마다 1분 증가 (시뮬레이션)
  - 사용 시간 표시 (00:00 형식)
  - 진행률 바 (%)로 시각화

- **시간 제한 규칙**
  - 일일 제한 시간 도달 시 자동 잠금
  - 잠금 시 경고 메시지 표시
  - 80% 도달 시 경고 알림
  - 초과 시 강제 세션 종료

- **세션 제어**
  - 앱 시작: `startShoppingAppSession()`
  - 앱 중지: `stopShoppingAppSession()`
  - 활성화된 앱만 시간 카운트
  - 앱 전환 시 이전 세션 자동 종료

### 2.3 사용 시간 시뮬레이션
- **수동 시간 조정**
  - 증가/감소 버튼 (분 단위)
  - 사용 시간 직접 입력 가능
  - 음수 방지 (0 이상만 허용)
  - 실시간 DB 반영

### 2.4 앱 잠금 기능
- **잠금 설정**
  - 앱별 잠금/잠금 해제 토글
  - 잠금 상태 시각적 표시
  - 잠금된 앱 접근 불가 (UI에서)
  
- **잠금 해제 조건**
  - 사용자의 수동 해제
  - 자정에 일일 사용 시간 리셋 (미구현 시 주석)

### 2.5 앱 수정 및 삭제
- **수정 기능**
  - 앱 정보 업데이트
  - 제한 시간 변경
  - 카테고리 변경
  
- **삭제 기능**
  - 앱 삭제 확인
  - 해당 앱의 모든 기록 정리

## 3. 결제 브레이크 (Payment Brake)

### 3.1 결제 브레이크 시뮬레이션
- **팝업 트리거**
  - 상품명 입력
  - 상품 가격 입력 (숫자만 필터링)
  - "결제 브레이크 시작" 버튼

### 3.2 다짐 시스템
- **템플릿 제공**
  - 한글 템플릿 5개
  - 영문 템플릿 5개
  - 선택 가능한 다짐 목록

- **다짐 템플릿 예시**
  - "진짜 필요한 것인지 최소 24시간 뒤에 다시 생각하자"
  - "이번 달 예산을 초과하면 다음 달의 내가 피눈물 흘리며 고통받는다"
  - "장바구니에만 담아두고 일주일 뒤에 냉정하게 다시 열어보자"
  - 등등 (총 5개)

- **커스텀 다짐**
  - 사용자가 직접 다짐 입력 가능
  - 템플릿 대신 커스텀 다짐 저장

### 3.3 신선한 명언 제시
- **팝업 표시 시 명언 갱신**
  - 결제 브레이크 팝업 오픈 시 자동으로 새로운 명언 로드
  - 사용자 격려 및 동기부여

### 3.4 결과 기록
- **차단 성공 (Blocked)**
  - "참기 성공" 버튼
  - 상품명, 가격, 선택한 다짐 저장
  - `isBlocked = true`로 기록
  - 성공 다이얼로그 표시

- **차단 실패 (Purchased)**
  - "지르기" 버튼
  - 상품명, 가격, 실패 사유 저장
  - `isBlocked = false`로 기록
  - "아쉬워... 다음 번에는 참을 수 있어요!" 다이얼로그

## 4. 통계 및 기록 관리

### 4.1 충동구매 기록 조회
- **기록 목록**
  - 전체 기록 표시
  - 최신순 정렬
  - 각 기록 항목
    - 상품명
    - 가격 (통화 표시)
    - 다짐 문구
    - 차단 여부 (✓ or ✗)
    - 기록 일시

### 4.2 통계 계산
- **집계 항목**
  - 총 차단 횟수
  - 총 구매 강행 횟수
  - 총 절약 금액 (차단된 상품의 합계)
  - 성공률 (차단 / 전체 × 100%)
  - 평균 상품 가격

### 4.3 기록 초기화
- **리셋 기능**
  - 모든 기록 삭제 확인 팝업
  - 데이터베이스 전체 정리
  - 통계 초기화

## 5. 사용자 설정 (Settings)

### 5.1 예산 관리
- **월간 예산 설정**
  - 기본값: 400,000원
  - 사용자 커스텀 설정 가능
  - 슬라이더 또는 직접 입력

### 5.2 알림 설정
- **예산 초과 알림**
  - 옵션: "예산 초과 시", "80% 도달 시" 등
  - 토스트/배너 알림 표시

- **야간 알림 제한**
  - 토글 스위치
  - 야간 시간대 알림 무음 처리

- **시간 제한 알림**
  - 앱별 시간 제한 알림 설정 ("1일 1시간", "1일 2시간" 등)

### 5.3 언어 설정
- **다국어 지원**
  - 한국어 / 영어 선택
  - 시스템 로케일 기반 자동 선택
  - 수동 언어 변경 저장

### 5.4 테마 설정
- **테마 선택**
  - Pastel (기본)
  - Cyberpunk
  - Sage
  - Yellow
  - Sky
  - 실시간 미리보기

## 6. UI/UX 요구사항

### 6.1 네비게이션
- **탭 기반 네비게이션**
  - 탭 0: 대시보드 (통계 + 명언)
  - 탭 1: 쇼핑앱 제어 (앱 관리 + 세션 제어)
  - 탭 2: 결제 브레이크 (시뮬레이터 + 기록)
  - iPhone 스타일의 하단 탭바

### 6.2 시각 디자인
- **색상 체계**
  - 테마별 컬러 팔레트
  - Pastel Blue, Peach, Mint, Yellow, Red 등

- **애니메이션**
  - 화면 전환 시 페이드 인/아웃
  - 버튼 터치 피드백
  - 리스트 항목 스크롤 애니메이션

- **반응형**
  - 모든 화면 크기 지원
  - 세로/가로 모드 대응

### 6.3 대화형 요소
- **모달/팝업**
  - 결제 브레이크 다이얼로그
  - 결과 다이얼로그
  - 확인 팝업 (삭제, 리셋 등)

- **경고/알림**
  - 상단 배너 (경고 메시지)
  - 토스트 메시지
  - 인라인 경고 (유효성 검사)

## 7. 데이터 관리

### 7.1 데이터 모델
- **ShoppingApp**
  - id (자동 증가)
  - name: 앱 이름
  - dailyLimitMinutes: 일일 사용 제한 시간
  - usedMinutesToday: 오늘 사용한 시간
  - isLocked: 잠금 여부
  - packageName: 패키지명
  - category: 카테고리

- **BlockedImpulse**
  - id (자동 증가)
  - itemName: 상품명
  - itemPrice: 상품 가격
  - resolutionSelected: 선택한 다짐
  - isBlocked: 차단 성공 여부
  - timestamp: 기록 일시

### 7.2 데이터 저장소
- **Room Database**
  - `ShoppingAppDao`: CRUD 작업
  - `BlockedImpulseDao`: 기록 저장/조회
  - 자동 마이그레이션 지원

### 7.3 상태 관리
- **ViewModel (ShoppingViewModel)**
  - StateFlow 기반 상태 관리
  - 데이터 흐름 중앙화
  - 라이프사이클 안전성

## 8. 기술 요구사항

### 8.1 아키텍처
- **MVVM 패턴**
  - ViewModel: 비즈니스 로직
  - Repository: 데이터 접근
  - UI (Composable): 화면 표시

- **비동기 처리**
  - Kotlin Coroutine
  - StateFlow / Flow
  - viewModelScope

### 8.2 라이브러리
- **Jetpack Compose**: UI 프레임워크
- **Room**: 로컬 데이터베이스
- **Kotlin Coroutines**: 비동기 작업
- **Material 3**: UI 컴포넌트

### 8.3 코드 품질
- **멀티스레드 안전성**
  - Mutex를 통한 동시성 제어
  - Race condition 방지
  
- **예외 처리**
  - try-catch 블록
  - graceful error handling
  
- **로깅**
  - 주요 이벤트 로깅
  - 디버깅 정보

## 9. 비기능 요구사항

### 9.1 성능
- 앱 시작 시간 < 2초
- 목록 스크롤 프레임드롭 없음 (60fps)
- 메모리 사용량 < 100MB

### 9.2 안정성
- 크래시 없는 안정적 동작
- 데이터 손실 방지
- 동시성 버그 방지

### 9.3 보안
- 로컬 데이터 암호화 (선택사항)
- 사용자 데이터 보호
- API 키 관리 (.env)

### 9.4 접근성
- 화면 리더 지원 (Accessibility)
- 명확한 텍스트 크기
- 높은 명도 대비
